# 教程 Markdown 写作规范（qtcraft）

这份规范用于后续所有教程文章（C++ / QML / Qt）。目标是：面向完全初学者，讲清楚“是什么 + 为什么 + 怎么用 + 容易错在哪 + 怎么练”，并且避免重复啰嗦。

---

## 1. 总原则

### 1.1 面向初学者，但术语要专业

- 第一次出现的概念：给出**专业名称（中英）** + **一句类比** + **一句话作用**。
- 之后再次出现：不再重复“是什么”，只写**新增规则/坑点/用法差异**，并用“（见 X.Y）”引用首次讲解位置。

推荐格式：

- 专业名称：XXX（English term）
- 类比：……
- 作用：……
- 规则/坑点：……

### 1.2 先给直觉，再给规则

顺序建议：

1. 先用一句话告诉读者“它解决什么问题”
2. 再给直觉类比（帮助记忆）
3. 再给可运行代码（让读者立刻看到结果）
4. 最后补规则与边界情况（避免新手踩坑）

### 1.3 每段只解决一个问题

- 一小节只讲一个知识点，不要把多个概念混在一起。
- 如果必须提到新概念：要么在本节首次讲全，要么明确说“先记住结论，后面章节讲”。

---

## 2. 文章结构（固定骨架）

每篇文章建议按下面骨架写，读者会形成固定预期：

1. 标题（编号 + 主题）
2. 目标（读完能做到什么）
3. 先决知识（可选：上一章需要会什么）
4. 核心内容（分小节）
5. 常见坑（集中列出）
6. 小练习（循序渐进）
7. 小结（你现在应该掌握什么）

---

## 2.1 目录与跳转（推荐）

- 每篇标题下面加一行：`目录页：见 [C++ 小白教程目录](00-目录.md)`（或对应目录）
- 文章结尾加“下一篇”链接，读者能顺滑往下学；如果你愿意，也可以加“上一篇”
- 链接尽量用相对路径，不要写本机绝对路径

---

## 3. 代码示例规范

### 3.1 新手友好：示例尽量“可直接运行”

- 引入新概念时，优先给**完整可编译的最小程序**（包含 `#include`、`int main()`、`return 0;`）。
- 如果只是演示一个表达式/语句，可给“片段”，但要在文中明确上下文条件（例如需要哪些头文件、变量类型）。

### 3.2 一段代码只服务一个目标

- 每个代码块都要回答一个问题，比如“如何输入”“如何判断”“如何循环”。
- 不要在同一个代码块里顺手引入太多新东西（例如：在讲 if 的同时又讲数组、指针）。

### 3.3 代码风格一致

- 命名：用能看懂的英文名（`score`、`sum`、`count`），不要用 `a/b/c` 糊弄。
- 输出：能让读者看懂结果，尽量带标签（`"sum=" << sum`）。
- 示例中避免“省略号代码”（例如 `...`），除非你在写进阶章节。

### 3.4 换行与输出约定（推荐）

- 默认换行用 `'\n'`，只有在“必须立刻刷新输出”的场景再用 `std::endl`
- 示例输出尽量可读：优先 `key=value` 或清晰的提示语

---

## 4. 术语与类比写法（标准化）

### 4.1 第一次出现：讲全

模板（复制即可）：

> - 专业名称：XXX（English term）
> - 类比：……
> - 作用：……
> - 规则/坑点：……

示例（写法示范）：

- 专业名称：命名空间（namespace）
- 类比：像“姓氏/门牌”，用 `std::` 表示“标准库这一家”
- 作用：避免不同库里同名符号冲突
- 规则/坑点：新手阶段优先老老实实写 `std::`，等熟了再讲 `using`

### 4.2 再次出现：不重复“是什么”

只写新增内容，并引用首次讲解：

- `std::cout`：这里不再解释（见 0.4），本节只补充 `'\n'` 与 `std::endl` 的差异：……

---

## 5. 常见坑写法

### 5.1 统一用“错误 → 结果 → 正确写法”

例：

- 错误：把比较写成赋值 `if (x = 3)`
- 结果：条件永远为真/行为异常
- 正确：`if (x == 3)`

### 5.2 坑要“能复现”

- 给一段能触发问题的输入或代码
- 解释为什么会这样（用读者能理解的语言）
- 给可直接替换的修正方案

---

## 6. 练习设计规范

### 6.1 每个练习固定包含 5 要素

- 题目：一句话说明要做什么
- 输入/输出：读什么、输出什么
- 约束：范围、类型、边界
- 提示：只给方向，不直接给答案
- 目标：练什么（对应本章知识点）

### 6.2 练习难度递进

建议每篇 4~6 题：

- 1~2 题：只练语法（能写出来就行）
- 1~2 题：练组合（两个知识点一起用）
- 1 题：综合小项目/小游戏（让读者爽一次）

### 6.3 不在正文里直接贴“完整答案”

新手会直接抄，学习效果很差。推荐二选一：

- 给“分步提示”（从思路到伪代码到关键代码片段）
- 或把完整答案放到单独的“题解”文章（后续需要时再补）

---

## 7. 文件命名与放置（推荐）

- 教程文件统一用编号前缀：`NN-主题.md`，NN 两位数字从 01 开始
- C++ 系列放在 `docs/cpp/`；QML/Qt 系列放在各自目录下（便于目录页聚合）
- 标题、文件名、目录页三者保持一致，避免读者找不到
