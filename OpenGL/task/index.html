    OpenGL 教程任务清单（从概念 → 窗口 → 三角形 → 逐步深入）¶

    本目录用于写一套“从零到能画出东西”的 OpenGL 教程，节奏强调：

    • 每章都能跑出可见结果（先让它亮起来，再谈优雅）
    • 每章都有：目标 → 最小实现 → 关键概念 → 常见坑 → 小练习
    • 代码尽量短、结构稳定、可复用（后面章节在同一骨架上迭代）
    • 行文偏“工程与直觉”，但数学/图形部分不回避关键推导

    说明：本教程会吸收优秀图形学写作的“问题拆解方式”和“直觉优先的解释顺序”，但正文与图示表述均保持原创；代码片段以“最小可运行”为标准组织。

    0. 总体目标与边界¶

    • 目标读者：会 C++ 基础、会 CMake 基础，想系统学 OpenGL（现代 OpenGL / Core Profile）
    • 目标平台：优先覆盖 Windows（后续补 Linux/macOS 差异）
    • OpenGL 版本：以 OpenGL 3.3 Core 为主线（足够通用、资料多）
    • 边界约定：
    • 不从固定管线讲起（直接用 shader 管线）
    • 不讲太多“API 列表”，强调“为什么要这样组织渲染代码”
    • 先做 2D/基础 3D 再进入 PBR/高级效果

    1. 教程目录规划（文件清单）¶

    这里只列计划文件名与主题，不创建文件；后续按任务逐篇落地。

    • 03-第一个三角形：VAO/VBO/EBO + 最小 Shader.md
    • 04-坐标与变换：NDC、矩阵、MVP、右手系.md
    • 05-颜色与插值：顶点属性、varying、Gamma 的直觉.md
    • 06-纹理：采样、wrap/filter、mipmap、sRGB.md
    • 07-相机：LookAt、轨道相机、第一人称相机.md
    • 08-深度与裁剪：depth test、face culling、clip space.md
    • 09-模型与网格：OBJ/GLTF（简化版）加载、法线、切线.md
    • 10-光照基础：Lambert/Phong/Blinn-Phong 与能量直觉.md
    • 11-阴影入门：Shadow Map 的最小实现与常见伪影.md
    • 12-离屏渲染：FBO、后处理、HDR/曝光.md
    • 13-性能与工程：状态切换、批处理、调试与 Profiling.md
    • 14-材质与 PBR 入门（可选）：BRDF 的直觉与实现骨架.md

    2. 写作规范（每章模板）¶

    • 标题：# NN-主题：一句话说明你将获得什么
    • 开头固定三段：
    • 目录页链接（回到 00-目录）
    • 本章目标（3~5 条）
    • 先决条件（明确依赖哪些前置章与工具）
    • 正文结构建议：
    • 1) 先跑起来（最小可见效果）
    • 2) 再解释“关键的 2~3 个概念”
    • 3) 最后做“结构化的工程改造”（让后续可扩展）
    • 每章必备：
    • 可复制代码块（含关键文件路径/片段）
    • 常见坑（至少 3 条，来自真实踩坑点：黑屏、坐标错、状态错、资源泄漏）
    • 小练习（至少 2 个：一个改参数、一个改结构）
    • 下一篇导航链接

    3. 里程碑任务（按“能看到结果”的顺序）¶

    Milestone A：能开窗、能打印 OpenGL 信息¶

    • 确认工具链：编译器、CMake、显卡驱动版本
    • 选择窗口与上下文库（主线推荐 GLFW；备选 SDL；也可补 Qt 版本）
    • 引入 OpenGL Loader（主线推荐 GLAD；备选 GLEW）
    • 打通最小工程：创建窗口 → 创建上下文 → 加载函数指针
    • 打印版本信息：OpenGL/GLSL/vendor/renderer
    • 加入调试输出：KHR_debug 回调 + glGetError 的使用边界

    交付标准： - 程序启动后窗口可见，不闪退 - 控制台打印出版本/显卡信息

    Milestone B：能清屏、能稳定刷新（告别“黑屏恐惧”）¶

    • 解释清屏发生在“帧循环”里（swap/poll/clear 的关系）
    • 做一个“颜色随时间变化”的清屏（验证循环与时间源）
    • 把窗口 resize 绑定到 viewport（避免拉伸/裁剪异常）

    交付标准： - 背景色可变化，resize 不崩且显示正常

    Milestone C：第一个三角形（现代 OpenGL 最小闭环）¶

    • 解释数据流：CPU 数组 → VBO → 顶点属性 → 顶点着色器 → 光栅化 → 片元着色器
    • 讲清 VAO 的意义（“把顶点输入的配置打包起来”）
    • 写最小 shader 编译/链接/报错输出
    • 画出三角形（glDrawArrays）
    • 加一个按键切换：wireframe / fill（验证状态机直觉）

    交付标准： - 三角形可见，shader 报错可读 - 能用按键切换线框

    Milestone D：把“能画”变成“能迭代”的工程骨架¶

    • 抽一个最小 Shader 类（load/compile/link/use/setUniform）
    • 抽一个最小 Mesh 结构（VAO/VBO/EBO 生命周期）
    • 约定资源目录结构（shaders/textures/models）
    • 约定错误处理策略（断言/日志/返回值）

    交付标准： - 新增一个物体只需要：给顶点数据 + 绑定 shader

    4. 深入路线（从“会画”到“会做效果”）¶

    阶段 1：坐标系与变换（你需要真正理解的第一堵墙）¶

    • NDC/Clip Space/Viewport 的区别与关系
    • 2D 变换（平移/旋转/缩放）与矩阵组合顺序
    • MVP 最小示例：让三角形绕自身旋转
    • 引入数学库（主线推荐 GLM）与“只用到哪讲到哪”

    阶段 2：纹理与采样（第二堵墙：为什么看起来“糊/花/黑”）¶

    • 纹理坐标与插值
    • wrap/filter/mipmap 的可视化实验
    • sRGB 与 Gamma 的直觉：为什么颜色会“发灰/发黑”

    阶段 3：深度、裁剪、剔除（第三堵墙：为什么会“穿模/闪烁/背面乱画”）¶

    • 深度缓冲的工作方式与精度问题
    • face culling 与 winding order
    • 近远平面与 z-fighting 的处理

    阶段 4：光照与材质（从“彩色物体”到“像真的”）¶

    • 法线、空间（模型/世界/视图）
    • 漫反射/高光的最小实现
    • 多光源与材质参数组织方式（uniform/UBO 的选择）

    阶段 5：阴影与后处理（工程化开始变难）¶

    • Shadow Map 最小闭环（一次渲染深度、一次正常渲染）
    • 解释常见伪影：Peter Panning、Shadow Acne、采样抖动
    • FBO 与后处理：灰度/模糊/色调映射

    5. 常见坑清单（写作时要反复强调）¶

    • 上下文没创建成功（版本/核心模式/驱动不支持）
    • OpenGL 函数指针没加载（loader 初始化顺序错）
    • viewport 没设置或 resize 没处理（导致看不到东西）
    • shader 编译失败但没打印 log（典型“黑屏”原因）
    • 顶点属性 stride/offset 错（典型“随机噪点三角形”）
    • 深度测试开关、清 depth buffer 漏掉（典型“穿模/残影”）
    • 资源生命周期混乱（VAO/VBO 删除时机不对）

    6. 后续协作约定（写代码时的统一口径）¶

    • 主线示例尽量保持“单 exe + 资源目录”即可跑
    • 章节之间复用同一套工程骨架（不反复换库/换风格）
    • 每次引入新抽象（类/模块）都要给出“为什么现在需要它”
